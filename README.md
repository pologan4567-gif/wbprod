# 🛒 WB Ad Manager Bot v2

Telegram-бот с полноценным **inline-интерфейсом** для управления рекламными кампаниями Wildberries.

**Что умеет:**
- 📋 Показывает **список всех твоих кампаний** с реальными статусами
- ▶️ / ⏸ Запускает и останавливает любую кампанию кнопкой
- ⏰ Хранит расписание для каждой кампании отдельно
- 🤖 Автоматически включает/выключает рекламу по cron (GitHub Actions)
- 📨 Присылает уведомление в Telegram после каждого действия

Работает **бесплатно** через GitHub Actions. Карта не нужна.

---

## 🚀 Быстрая настройка

### 1. Подготовь данные

| Что нужно | Где взять |
|---|---|
| `TELEGRAM_TOKEN` | [@BotFather](https://t.me/BotFather) → /newbot |
| `ALLOWED_USER_ID` | [@userinfobot](https://t.me/userinfobot) — напиши боту |
| `WB_API_TOKEN` | seller.wildberries.ru → Настройки → Доступ к API → Создать токен (тип: **Реклама**) |
| `CAMPAIGN_IDS` | ID кампаний через запятую (найдёшь через бота командой /list или в WB личном кабинете) |

### 2. Создай репозиторий на GitHub

1. [github.com](https://github.com) → **New repository** → назови `wb-ad-manager` → **Private** → Create
2. Загрузи все файлы через **Add file → Upload files**
3. Папку `.github/workflows/` создай через **Add file → Create new file**, введи путь:
   - `.github/workflows/wb-bot.yml` → вставь содержимое файла
   - `.github/workflows/wb-scheduler.yml` → вставь содержимое файла

### 3. Добавь секреты

**Settings → Secrets and variables → Actions → New repository secret**

| Секрет | Значение |
|---|---|
| `TELEGRAM_TOKEN` | Токен бота |
| `ALLOWED_USER_ID` | Твой Telegram ID |
| `WB_API_TOKEN` | Рекламный токен WB |
| `CAMPAIGN_IDS` | ID кампаний через запятую: `12345678,87654321` |

### 4. Запусти бота

**Actions → WB Telegram Bot (polling) → Run workflow**

Через ~30 секунд напиши боту `/start` — он ответит!

## ⏰ Как работает автоматическое расписание

Два независимых механизма:

### Автоматический (GitHub Actions cron)
GitHub запускает `cron_action.py` строго по расписанию — без бота:
- `0 5 * * *` = **08:00 МСК** → все кампании из `CAMPAIGN_IDS` запускаются
- `0 20 * * *` = **23:00 МСК** → все кампании останавливаются

### Изменить время расписания
Открой `.github/workflows/wb-scheduler.yml` и измени строки:
```yaml
- cron: '0 5 * * *'   # 08:00 МСК (UTC+3 → UTC: час минус 3)
- cron: '0 20 * * *'  # 23:00 МСК
```

Таблица перевода МСК → UTC:
| МСК | UTC (пишем в cron) |
|---|---|
| 07:00 | `0 4 * * *` |
| 08:00 | `0 5 * * *` |
| 09:00 | `0 6 * * *` |
| 21:00 | `0 18 * * *` |
| 22:00 | `0 19 * * *` |
| 23:00 | `0 20 * * *` |

## 🔄 Как работает Telegram-бот на GitHub Actions

Бот запускается в режиме polling как GitHub Actions job (timeout 350 минут).
Каждые 6 часов cron перезапускает его (GitHub Actions лимит = 6 часов на job).
Настройка `concurrency` гарантирует что работает только одна копия бота.

**⚠️ Важно:** При перезапуске бота настройки расписания в боте сбрасываются
(они хранятся в памяти). Автоматический cron работает независимо — он не зависит
от состояния бота.
